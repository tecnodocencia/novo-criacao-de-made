Implementadas 5 melhorias na sessão de 2026-07-01.

**Fix #1 — Bug: Nome do Autor não salva**
Raiz: `salvarJogo()` em `js/database.js` retornava `data[0]` sem remap camelCase. `state.games[idx]` ficava com `disciplineinfo` (lowercase) em vez de `disciplineInfo`, então o dashboard mostrava autores como "-" imediatamente após salvar (mesmo que os dados estivessem corretos no banco após `listarJogos` mapear).
Fix: extraída função `remapJogo(jogo)` usada em `listarJogos()`, `salvarJogo()` e `obterJogoPorCodigo()`. Também adicionado `this.persistEditorFields()` no início de `saveGame()` em `editorShell.js`.

**Fix #2 — Editor de caracteres especiais**
Projeto é vanilla JS sem npm. Não é React. Expandido o painel existente em `index.html` (#modal-card) com 8 categorias via sistema de abas: Matemática, Cálculo (∫,∂,∑,∏,∇), Química (setas de reação), Frações Unicode (½,⅓,¼,⅔,¾, etc.), Grego (completo min/mai), Conjuntos/Lógica (∈,∉,∩,∪,∅,∀,∃), Geometria (∠,⊥,∥), Sub/Superscripts. Função `switchSymTab(tabId)` adicionada em `utils.js`. Classe CSS `.sym-btn` adicionada ao `<style>` do `index.html`.

**Fix #3 — Cartas escondidas (overflow)**
`overflow-hidden` no container centro de `partials/games/codigo-secreto/player.html` (linha 27) cortava cartas horizontalmente quando havia 5-6 cartas. Fix: substituído por `min-w-0` (sem overflow-hidden). Também adicionado `flex-wrap justify-center` nos containers `#play-secret-slots` e `#play-drop-slots` para wrap automático.

**Feature #4 — Link público para jogar**
Novo botão "Compartilhar" (azul, ícone fa-share-nodes) em cada card do dashboard → `js/core/dashboard.js`.
Novo método `shareGame(gameId)` em `dashboardMethods`: gera `share_code` de 6 chars via `dbService.gerarCodigoCompartilhamento()` e abre `#modal-share` com URL completa.
Novo modal `#modal-share` adicionado ao final de `index.html` (antes de `<script>`).
Métodos: `shareGame`, `copyShareUrl`, `closeShareModal` em `dashboardMethods`.
Handler de click-fora adicionado em `app.js`.
Novos métodos em `database.js`: `gerarCodigoCompartilhamento`, `obterJogoPorCodigo`, `salvarResultadoPublico`, `listarRankingJogo`.

**Feature #5 — Sistema de ranking**
Novos arquivos: `play.html` (root) e `js/play.js`.
`play.js` é completamente standalone: importa só `./supabase.js` e `./games/codigo-secreto/model.js`. NÃO depende de `app.js`, `editorShell.js`, `playerMethods`, etc.
Fluxo: Welcome → Difficulty selection → Game (click+drag) → Solution modal com flip animation → Ranking screen.
Score = `(rules.attempts - attemptsUsed) * 100 + codeSize * 10` para vitórias, 0 para derrotas.
Ranking carregado da tabela `public_plays` (top 20, só vitórias, score DESC, played_at ASC).

**SQL necessário**: `supabase-migrations.sql` na raiz do projeto — precisa ser executado pelo desenvolvedor no Supabase SQL Editor.

**Why:** Feature pedida para uso pedagógico — professores querem compartilhar jogos com alunos sem que eles precisem de conta.
**How to apply:** Sempre que tocar em compartilhamento, verificar que `share_code` é preservado em `salvarJogo()` payload. O `play.js` é standalone — não quebrar por mudanças em `app.js`.
